### 4.4 日志配置

项目使用 Logback 作为日志框架，配置文件位于 `src/main/resources/logback-spring.xml`。

**日志级别**:
- **ERROR**: 错误信息（严重问题）
- **WARN**: 警告信息（潜在问题）
- **INFO**: 一般信息（默认级别）
- **DEBUG**: 调试信息（开发时使用）
- **TRACE**: 跟踪信息（详细调试）

**修改日志级别**:

在 `application.yml` 中配置:
```yaml
logging:
  level:
    root: INFO
    com.example.demo: DEBUG  # 只对特定包开启 DEBUG
```

**日志文件位置**:
- 应用日志: `logs/app.log`
- 错误日志: `logs/error.log`

**查看日志**:
```bash
# 实时查看应用日志
tail -f logs/app.log

# 搜索错误日志
grep "ERROR" logs/app.log
```
```

**补充原因**:
根据项目分析报告第5.4节，项目配置文件中包含日志配置，且存在 `logs/` 目录。原文档未说明日志配置，补充此内容以帮助用户调试和排查问题。
