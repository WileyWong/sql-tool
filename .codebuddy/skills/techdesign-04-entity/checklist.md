# 实体设计检查清单

## 设计完整性
- [ ] 所有核心业务概念已识别为实体或值对象
- [ ] 实体和值对象区分清晰
- [ ] 每个实体有唯一标识（ID）
- [ ] 实体不超过 50 个字段

## 关系设计
- [ ] 优先使用单向关系 + ID 引用
- [ ] 聚合内部使用对象引用 + 级联操作
- [ ] 跨聚合使用 ID 引用
- [ ] 避免大量双向 `@OneToMany`/`@ManyToOne`

## 充血模型
- [ ] 业务逻辑在实体内部，不在 Service 层
- [ ] 使用工厂方法创建实体
- [ ] 不变量验证完整
- [ ] 无参构造函数是 `protected`

## 技术实现
- [ ] 金额用 `BigDecimal`
- [ ] 枚举用 `EnumType.STRING`
- [ ] 审计字段完整（createdAt、updatedAt）
- [ ] 索引设计合理

## 🚨 红灯信号
以下任一出现，立即停止：
- [ ] 实体只有 getter/setter，无业务方法
- [ ] 单个实体超过 50 个字段
- [ ] 可以创建无效对象（如负数金额）
