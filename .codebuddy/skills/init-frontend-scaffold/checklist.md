# 初始化前端脚手架 - 验证清单

## 📋 完成检查清单

使用此清单验证项目初始化是否完成。

### 选择初始化方式

- [ ] **手动执行** - 按照步骤 1-8 手动执行
- [ ] **脚本执行** - 使用自动化脚本（跳过步骤 1-7，直接进行步骤 8）

---

## 📋 手动执行清单

如果选择手动执行，请按照以下步骤：

### 阶段 1: 项目信息收集

- [ ] 项目名称已确定（英文，小写字母 + 连字符）
- [ ] 项目简介已准备（1-2 句话）
- [ ] 项目版本已确定（通常为 0.1.0）
- [ ] 前端框架已选择（Vue 3等）
- [ ] 主要依赖库已列出
- [ ] 作者/团队信息已准备
- [ ] 项目输出位置已确定

### 阶段 2: 脚手架克隆

- [ ] Git 已安装并配置
- [ ] 有访问脚手架仓库的权限
- [ ] 克隆命令执行成功
- [ ] 项目目录已创建
- [ ] 所有脚手架文件已克隆
- [ ] `.git` 目录存在（包含脚手架历史）
- [ ] 没有克隆错误或警告

**验证命令**:
```bash
ls -la {project-name}
git log --oneline | head -5
```

### 阶段 3: Git 依赖移除

- [ ] `.git` 目录已删除
- [ ] 新的 `.git` 目录已创建
- [ ] Git 用户信息已配置
- [ ] 所有文件已添加到暂存区
- [ ] 初始提交已完成
- [ ] 提交信息为 "chore: initial commit from scaffold"
- [ ] `git log` 显示只有一个提交

**验证命令**:
```bash
git log --oneline
# 应该显示: abc123 chore: initial commit from scaffold

git status
# 应该显示: On branch main, nothing to commit, working tree clean
```

### 阶段 4: 项目配置改造

#### package.json 修改

- [ ] `name` 字段已更新为项目名称
- [ ] `version` 字段已更新为初始版本
- [ ] `description` 字段已更新为项目简介
- [ ] `author` 字段已更新为作者信息
- [ ] `license` 字段已更新为许可证类型
- [ ] JSON 格式正确（无语法错误）
- [ ] 所有必要的依赖都存在

**验证命令**:
```bash
cat package.json | jq .
# 检查 JSON 格式是否正确

npm list
# 检查依赖是否完整
```

#### README.md 修改

- [ ] 项目标题已更新
- [ ] 项目描述已更新
- [ ] 快速开始指南已更新
- [ ] 项目结构说明已更新
- [ ] 开发规范已说明
- [ ] 许可证信息已更新
- [ ] Markdown 格式正确

**验证命令**:
```bash
head -20 README.md
# 检查标题和描述是否正确
```

#### 其他配置文件修改

- [ ] `tsconfig.json` 已检查（如需要）
- [ ] `vite.config.ts` 已检查（如需要）
- [ ] `.env.example` 已创建（如需要）
- [ ] `public/index.html` 已更新（如需要）

### 阶段 5: 项目内容改造

#### 项目结构调整

- [ ] 不需要的示例代码已删除
- [ ] 新的目录结构已创建
- [ ] 模块目录已重命名（如需要）
- [ ] 文件组织结构已调整（如需要）

**验证命令**:
```bash
tree src/
# 检查项目结构是否符合预期
```

#### 源代码修改

- [ ] 组件名称已更新（如需要）
- [ ] 导入路径已更新
- [ ] 业务逻辑已调整（如需要）
- [ ] 样式和主题已更新（如需要）
- [ ] 没有编译错误

**验证命令**:
```bash
npm run build
# 检查是否有编译错误
```

#### 资源文件替换

- [ ] Logo 已替换（如需要）
- [ ] 图标已替换（如需要）
- [ ] 静态资源已更新（如需要）
- [ ] 主题配置已更新（如需要）

### 阶段 6: 依赖安装

- [ ] Node.js 版本满足要求（>= 16.0.0）
- [ ] npm 版本满足要求（>= 8.0.0）
- [ ] `npm install` 执行成功
- [ ] 没有依赖冲突
- [ ] `node_modules` 目录已创建
- [ ] `package-lock.json` 已生成
- [ ] 所有依赖都已安装

**验证命令**:
```bash
node --version
npm --version
npm list
# 检查依赖是否完整
```

### 阶段 7: 项目启动验证

- [ ] `npm run dev` 执行成功
- [ ] 开发服务器启动成功
- [ ] 没有启动错误
- [ ] 浏览器能访问项目（http://localhost:5173）
- [ ] 项目页面正常显示
- [ ] 没有控制台错误
- [ ] 没有明显的警告

**验证命令**:
```bash
npm run dev
# 在浏览器中访问 http://localhost:5173
```

### 阶段 8: 项目完整性检查

- [ ] 所有改造都不影响项目启动
- [ ] 项目功能正常
- [ ] 没有破坏脚手架的核心功能
- [ ] 代码遵循项目规范
- [ ] 没有拼写错误
- [ ] 没有格式错误
