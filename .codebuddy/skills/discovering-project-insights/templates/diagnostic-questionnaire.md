# 诊断问卷模板

用于采集专家知识和业务背景

## 📋 第一轮: 基本情况

### Q1: 项目基本信息

**问题**: "能简要介绍一下项目情况吗?"

**回答框架**:
- 项目名称和类型 (Web/Mobile/API/etc)
- 项目规模 (团队大小、代码量)
- 项目年龄和当前阶段
- 日活用户量 / 日均流量

---

### Q2: 项目阶段

**问题**: "项目属于哪个发展阶段?"

**选项**:
- [ ] 初创期 (<3 个月)
- [ ] 快速发展期 (3-12 个月)
- [ ] 成熟稳定期 (1-3 年)
- [ ] 维护期 (>3 年)

**后续问题** (根据答案):
- 如果初创期: "融资压力大吗? 是否需要快速扩展?"
- 如果成熟期: "是否有大规模重构计划?"
- 如果维护期: "是否考虑现代化改造?"

---

### Q3: 核心业务

**问题**: "项目的核心业务是什么?"

**示例**:
- 电商平台 (用户、商品、订单、支付)
- SaaS 服务 (多租户、账户、工作流)
- 内容平台 (用户、内容、推荐、社交)

---

## 📋 第二轮: 当前困境

### Q4: 主要痛点

**问题**: "目前遇到的最大问题是什么?" (多选)

**选项**:
- [ ] 代码质量差，bug 多
- [ ] 系统性能不足
- [ ] 功能开发速度慢
- [ ] 难以维护和扩展
- [ ] 部署和运维困难

- [ ] 技术栈过时

**后续问题**:
- "哪个痛点影响最大? 为什么?"
- "对业务有什么具体影响?"
- "已有哪些改进尝试?"

---

### Q5: 技术债规模评估

**问题**: "您认为项目的技术债有多严重?"

**选项**:
- [ ] 很轻微 (基本没有)
- [ ] 可控 (有一些，但可管理)
- [ ] 中等 (开始影响开发效率)
- [ ] 严重 (严重拖累开发)
- [ ] 危机 (急需处理)

**后续问题**:
- "技术债主要在哪些方面?" (架构/代码/基础设施)
- "对交付周期的影响多大?"

---

### Q6: 当前质量度量

**问题**: "你了解项目的质量指标吗?"

**选项**:
- [ ] 有完整的质量监控
- [ ] 有部分指标 (如代码覆盖率)
- [ ] 很少有量化指标
- [ ] 完全没有

**如果有指标，请补充**:
- 代码覆盖率: _______% 
- 缺陷密度: _______ bugs/KLOC
- 平均修复时间: _______ 小时
- 自动化测试比例: _______%

## 📋 问卷使用指南

### 分阶段使用

**初次访谈** (1 小时):
- Q1-Q6: 基本情况和痛点
- Q13-Q15: 需求和规划

**深度诊断** (2-3 小时):
- 所有问卷题目
- 根据回答进行追问

**定向诊断** (30 分钟):
- 只问卷相关题目
- 快速了解特定方面

### 访谈技巧

1. **分批提问**: 一次 2-3 个，避免认知过载
2. **避免是非题**: 使用开放式问题深入探讨
3. **记录要点**: 便于后续分析
4. **追问深入**: 理解问题背后的原因
5. **确认理解**: "我的理解是... 对吗?"

### 输出整理

将访谈结果转换为:
- [ ] 问题清单
- [ ] 优先级排序
- [ ] 改进方案
- [ ] 资源预估
- [ ] 时间计划
