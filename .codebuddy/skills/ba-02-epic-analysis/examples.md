## 简单案例：智能访客管理系统

### 输入材料

> 以下输入来自 ba-01 业务需求提取的输出（已转换为 Epic 分析所需格式）

```
【Epic名称】智能访客管理系统

【业务背景】
某大型企业园区每日访客量200+，现有纸质登记方式效率低、安全风险高。
需要建设一套智能访客管理系统，实现访客预约、审批、签到、离开全流程线上化。

【核心目标】
提升访客管理效率：
- 访客预约处理时间从平均30分钟降低到5分钟
- 安保人员现场确认时间从10分钟降低到2分钟
- 系统自动审批率达到60%以上

【主要功能点】
- 访客在线预约
- 被访人审批
- 访客签到/签退
- 访客数据统计

【涉及用户角色】
- 访客（外部来访人员）
- 被访人（企业内部员工）
- 安保人员
- 系统管理员

【顶层业务用例】（来自 ba-01 输出）
业务用例1：管理访客预约申请
- 主要参与者：访客（外部来访人员）
- 利益相关者：被访人、安保人员、企业管理层
- 核心流程：访客填写预约信息 → 系统验证 → 推送审批通知
- 业务规则：同一访客每天只能提交1次预约；访问日期范围今天～今天+30天

【非功能需求】
- 性能：500并发用户，页面加载P95<3秒
- 安全：身份证号AES-256加密，数据保留30天后自动删除
- 集成：微信公众号、门禁系统

【技术约束】
- 需对接微信公众号（需申请开发资质）
- 需对接现有门禁系统
```

## 第4部分：拆解策略

**推荐拆解维度**: 按工作流/场景

**理由**: 
- 访客管理是典型的端到端业务流程（预约→审批→签到→离开）
- 按工作流拆解能确保每个阶段独立可交付、用户体验完整

| 拆解分组 | 所属 Feature | 包含内容 | 预估故事数 | 优先级 |
|---------|-------------|---------|-----------|--------|
| 预约阶段 | F001 | 填写信息、上传证件、提交申请 | 4 | Must |
| 审批阶段 | F002 | 审批通知、审批操作、自动审批 | 3 | Must |
| 签到阶段 | F003 | 扫码签到、身份核验、访客通行 | 3 | Must |
| 离开阶段 | F003 | 签退记录、超时提醒 | 1 | Must |
| 数据统计 | F004 | 访客报表、数据导出 | 2 | Should |
| 黑名单管理 | F005 | 黑名单维护、自动拦截 | 2 | Could |

**总体预估**:
- Feature 总数: 5个
- 预估用户故事总数: 15个
- 建议迭代数: 3个迭代（按2周/迭代）
- 核心功能占比: 73%（Must Have 11个故事）

## 第5部分：MVP 定义

|------|------|
| MVP 范围 | F001 + F002 + F003（共11个Story） |
| 核心假设 | 微信公众号是访客主要入口；被访人及时审批率高 |
| 验证指标 | 预约成功率≥90%；审批响应时间<30分钟；签到核验时间<2分钟 |
| 预计交付 | 2个迭代（1个月） |

**MVP后迭代规划**:

| 迭代 | 范围 | 预计时间 |
|------|------|---------|
| MVP | 预约+审批+签到签退 | 1个月 |
| R1.1 | 数据统计 + 黑名单 | 2周 |
| R1.2 | 人脸识别 + 高级报表 | 2周 |

```
【Epic名称】全球化员工入职管理平台

【业务背景】
某全球化互联网头部企业（员工10万+，业务覆盖30+国家/地区）需要建设统一的入职管理平台。

- HR运营模式：三支柱模型（SSC + COE + HRBP）

【核心目标】
建设全球统一入职管理平台：
- 支持12种员工类型差异化入职流程
- 支持6大BG + 30+BU + 50+海外实体的组织架构
- 实现集团管控与区域自治的平衡
- 入职准备时间从5天缩短至1天
- 校招季批量入职处理能力：1000人/天
- 新员工满意度提升至95%+

- 多语言支持

- 新员工（正式/校招/实习/返聘/高管/博士后/兼职）

- 多语言：8种语言支持，日期/货币/地址本地化
- 高并发：5000并发用户，1000人/天批量入职
- 安全合规：等保三级、SOC2、ISO27001
- 系统集成：20+核心系统对接

【技术约束】
- 多租户架构，50+法律实体数据物理隔离
- 全球部署，多国数据本地化存储
```

- **F002**: 表单加载时间<3秒；字段配置变更无需开发介入
- **F003**: 审批链生成正确率100%；支持6维度规则配置
- **F004**: 资源分配自动化率≥80%；分配失败时自动告警
- **F005**: 供应商可自主提交人员信息；SSC审核通过率可追踪
- **F006**: 覆盖中国、美国、欧盟、日本、新加坡等主要国家合规要求
- **F007**: 支持1000人/天批量导入；批量审批支持分组处理
- **F008**: 规则变更生效时间<4小时；支持规则版本回滚
- **F009**: 审计日志保留7年；支持按实体导出合规报表
- **F010**: 支持8种语言；语言切换响应时间<1秒
- **F011**: 核心系统数据同步延迟<5分钟；接口可用性99.9%
- **F012**: 高管入职全流程保密；支持董事会审批流程
