- 标注条件显示的组件（如低数据提示）

## 质量要求

### 可追溯性（强制）
- 每个页面标注关联的用户旅程阶段
- 每个组件标注关联的用户故事编号
- 格式：`（来自US-01）` 或 `（对应旅程阶段：系统选择）`

### 完整性（强制）
- 页面结构需覆盖用户旅程100%的阶段
- 组件清单需覆盖用户故事100%的功能点
- 字段定义需包含所有需要收集的数据

### 规范性（强制）
- 组件命名使用驼峰命名法（如：systemSearchBox）
- 字段类型使用标准类型（String/Number/Boolean/Enum/Text/Date）
- 验证规则使用明确的表达式或条件

## 组件状态定义

每个交互组件需说明以下状态：
- 默认状态
- 交互状态（hover/active/focus）
- 特殊状态（disabled/loading/error）

## 信息不足处理

1. **合理推断**: 基于行业最佳实践补充
2. **明确标注**: `（推断，待确认）`
3. **列入待办**: 在"设计约束与建议"中标注待确认项
