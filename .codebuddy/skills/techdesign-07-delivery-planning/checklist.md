# 任务规划检查清单

## 技术选型验证
- [ ] 技术选型有明确理由（团队能力+需求适配+长期维护）
- [ ] 技术栈版本明确（如 Spring Boot 3.2.x）
- [ ] 架构模式明确（单体/分层/微服务）
- [ ] 技术选型决策矩阵完整

## 模块划分验证
- [ ] 所有模块功能定义明确
- [ ] 所有模块有清晰的API接口或组件列表
- [ ] 模块依赖关系清晰，无循环依赖
- [ ] 模块划分符合单一职责原则

## 任务拆解验证
- [ ] 所有任务粒度合理（1-3天）
- [ ] 所有任务有明确验收标准
- [ ] 所有任务依赖关系明确
- [ ] 所有任务有优先级标记（P0-P3）
- [ ] 关键路径已识别

## 工作量评估验证
- [ ] 总工作量评估合理（含20%风险缓冲）
- [ ] 分阶段工作量评估详细
- [ ] 里程碑计划明确（时间节点、交付物、验收标准）
- [ ] 风险评估完整（影响、概率、应对措施）
- [ ] 使用科学估算方法（三点估算）

## 文档质量验证
- [ ] 技术方案文档完整（技术选型、架构设计、模块划分）
- [ ] 实施计划文档完整（任务拆解、工作量评估、风险管理）
- [ ] 文档符合输出目录规范
- [ ] 文档引用项目记忆系统

## 🚨 红灯信号
以下任一出现，立即停止：
- [ ] 技术选型无依据（只说"流行"）
- [ ] 任务粒度过粗（>5天）
- [ ] 无验收标准
- [ ] 工作量估算过于乐观（无风险缓冲）
- [ ] 任务无依赖关系标注
