# 任务规划示例

## 示例 1: 用户管理系统技术方案

### 场景
开发一个用户管理系统，包含登录、注册、权限管理。

### 技术选型

```markdown
## 技术选型决策

### 后端框架: Spring Boot 3.2
- **团队能力**: 团队80%成员熟悉，降低学习成本
- **需求适配**: 支持RESTful API、JWT认证、数据库事务
- **长期维护**: 活跃社区，LTS版本，降低风险

### 前端框架: Vue 3
- **团队能力**: 团队全员熟悉Vue 3，Composition API 经验丰富
- **需求适配**: 支持复杂交互、组件复用、响应式系统强大
- **长期维护**: 社区活跃，生态丰富，官方支持完善

### 数据库: MySQL 8.0
- **团队能力**: 团队熟悉MySQL
- **需求适配**: 支持事务、索引，性能满足
- **长期维护**: 成熟稳定，广泛应用
```

### 模块划分

```markdown
## 后端模块

### 1. 用户模块
- **功能**: 用户认证、用户管理、权限控制
- **API**: POST /auth/login, GET /users/:id
- **数据表**: users, roles, permissions

### 2. 基础模块
- **功能**: 日志、异常处理、配置管理
```

### 任务拆解

```markdown
## Phase 1: 基础搭建 (第1周)

### Task 1.1: 项目初始化
- **工作量**: 1人天
- **子任务**:
  - [ ] 创建Spring Boot项目
  - [ ] 创建Vue 3项目（Vite+TypeScript）
  - [ ] 配置代码规范（ESLint+Prettier）
- **验收标准**:
  - [ ] 项目可正常启动
  - [ ] 代码规范检查通过

### Task 1.2: 数据库设计
- **工作量**: 2人天
- **子任务**:
  - [ ] 设计ER图（users, roles, permissions）
  - [ ] 编写DDL脚本
- **验收标准**:
  - [ ] 数据表创建成功
  - [ ] 符合数据库字段规范

## Phase 2: 核心功能 (第2-3周)

### Task 2.1: 用户认证
- **工作量**: 3人天
- **子任务**:
  - [ ] 实现登录接口
  - [ ] 实现JWT验证
  - [ ] 实现登录页面
- **依赖**: Task 1.1, Task 1.2
- **验收标准**:
  - [ ] 用户可登录
  - [ ] Token验证正常
  - [ ] 单元测试覆盖率≥80%
```

### 工作量评估

```markdown
## 总体评估
- 总工作量: 15人天
- 开发周期: 3周
- 团队规模: 2人（1前端+1后端）
- 风险缓冲: 3人天（20%）
- 最终工作量: 18人天

## 里程碑
| 里程碑 | 时间 | 交付物 |
|--------|------|--------|
| M1: 基础完成 | 第1周末 | 项目框架 |
| M2: 功能完成 | 第2周末 | 用户认证 |
| M3: 测试完成 | 第3周末 | 测试报告 |
```

## 示例 2: 电商订单系统任务拆解

### 场景
开发订单管理功能，包含订单创建、支付、发货。

### 任务拆解

```markdown
### Task 2.1: 订单创建
- **工作量**: 2人天
- **子任务**:
  - [ ] 实现创建订单接口
  - [ ] 实现订单列表接口
  - [ ] 实现订单详情页面
- **依赖**: 用户认证模块
- **优先级**: P0
- **验收标准**:
  - [ ] 用户可创建订单
  - [ ] 订单数据正确保存
  - [ ] 单元测试通过

### Task 2.2: 订单支付
- **工作量**: 3人天（含第三方对接）
- **子任务**:
  - [ ] 对接支付宝/微信支付
  - [ ] 实现支付回调
  - [ ] 实现支付状态更新
- **依赖**: Task 2.1
- **优先级**: P0
- **验收标准**:
  - [ ] 支付流程完整
  - [ ] 支付状态同步正确
  - [ ] 支持退款

### 三点估算示例

**Task 2.2 工作量评估**:
- 乐观值: 2人天（一切顺利）
- 正常值: 3人天（正常情况）
- 悲观值: 5人天（遇到对接问题）
- 期望工作量: (2 + 4×3 + 5) / 6 = 3.2人天
- 最终评估: 3人天
```

### 风险评估

```markdown
| 风险项 | 影响 | 概率 | 应对措施 | 缓冲 |
|--------|------|------|---------|------|
| 支付对接延迟 | 高 | 中 | 提前对接测试环境 | 1人天 |
| 并发性能问题 | 中 | 中 | 提前压测，优化 | 1人天 |
```

---

## 关键点总结

### 技术选型
- 始终考虑团队能力、需求适配、长期维护三要素
- 提供决策依据，不只是"因为流行"

### 任务拆解
- 粒度：1-3天
- 验收：明确标准
- 依赖：清晰标注

### 工作量评估
- 使用三点估算
- 预留20%风险缓冲
- 识别关键路径
