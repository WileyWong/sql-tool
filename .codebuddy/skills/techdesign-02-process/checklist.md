# 业务流程设计检查清单

## 核心业务对象
- [ ] 业务对象已识别（≥2个）
- [ ] 每个对象生命周期已明确（≥3个阶段）
- [ ] 每个对象关键事件已列出（≥3个）
- [ ] 业务约束已定义

## 状态机设计
- [ ] 所有状态已定义（≥5个）
- [ ] 所有状态转移已明确（≥8个转移）
- [ ] 每个转移包含：触发事件、前置条件、转移动作、后置条件
- [ ] 禁止的转移已明确列出
- [ ] 时间约束已定义
- [ ] 状态转移图清晰可读

## 主要业务流程
- [ ] 主要流程已识别（≥3个）
- [ ] 每个流程阶段已明确（≥3个阶段）
- [ ] 每个阶段步骤已详细描述
- [ ] 流程参与者已明确
- [ ] 异常流程已识别（≥3个）
- [ ] 流程中无技术细节（无API、数据库、UI）

## 业务规则
- [ ] 计算规则已定义（≥3个）
- [ ] 决策规则已定义（≥5个）
- [ ] 验证规则已定义（≥3个）
- [ ] 每个规则都有明确条件和结果
- [ ] 规则都是可量化的（无"尽快"、"尽量"等模糊词）

## 角色权限
- [ ] 系统角色已定义（≥3个）
- [ ] 每个角色职责已明确
- [ ] 关键操作权限已定义（≥10个）
- [ ] 权限边界已明确
- [ ] 权限隔离规则已定义

## 异常处理
- [ ] 主要业务异常已识别（≥10个）
- [ ] 每个异常都有明确处理方案
- [ ] 补偿机制已设计（≥3个）
- [ ] 重试机制已设计（≥3个）
- [ ] 幂等性处理已考虑

## 文档质量
- [ ] 文档结构清晰
- [ ] 内容完整无遗漏
- [ ] 无技术实现细节
- [ ] 符合输出目录规范

## 🚨 红灯信号
以下任一出现，立即停止：
- [ ] 流程中包含技术细节（如API接口、数据库表）
- [ ] 状态机不完整（遗漏重要状态或转移）
- [ ] 业务规则模糊（使用"尽快"、"尽量"等词）
- [ ] 缺少异常处理（<10个异常）
- [ ] 无权限边界定义
- [ ] 时间约束未量化
